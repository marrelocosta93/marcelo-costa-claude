# Protocolo GSD — Get Shit Done

## Princípios
1. **Comece pelo mais arriscado** — fail fast, feedback rápido
2. **Progresso visível** — reporte cada arquivo criado/modificado
3. **Viés para ação** — decisão técnica menor: decida sozinho. Arquitetural: proponha e peça OK
4. **Tarefas atômicas** — cada passo produz algo que compila
5. **Quick Mode** — tarefas < 30 min com escopo claro: pule plano, implemente direto
6. **Commits incrementais** — commite conforme avança, não só no final
